# 提交说明版短摘要

## 建议摘要

- 自选股：补齐港股实时展示与涨跌幅、优化标签管理与快捷编辑、支持删除联动清理
- 数据同步：新增同步历史与数据清理能力，支持按类型查看/删除
- 内容数据：新增“内容数据”浏览入口，支持查看已同步新闻、互动问答、社媒热度与新闻回退，并联动同步历史和数据清理
- 社媒分析：A 股原生社媒同步扩展到官方互动、东方财富热度、雪球热度、百度热搜，分析前预同步优先要求原生社媒
- 社媒主链路：A 股原生社媒同步现在只保留原生一手互动/热度源，新闻回退改为独立显式入口
- 同步入口：在“多数据源同步”页新增 A 股内容同步卡片，可直接触发原生社媒同步和新闻回退同步
- 行情兜底：A 股在线行情补齐接入独立 `sina_finance` 最后兜底，并回传交易日和更新时间字段
- 社媒口径：A 股社媒分析报告新增来源分布、原生样本数和“仅新闻回退”提示
- 港股新闻：港股原生新闻链路新增数据源尝试明细，原生源失败时可直接看到 `akshare / finnhub` 状态
- 数据源：完善单股同步的实时/非实时/混合路由规则
- 配置管理：统一厂家/数据源密钥来源展示与新增保存逻辑
- 分析报告：增强中文报告标题规范与长文本阶段稳定性
- 工程化：整理公开部署脚本、提交检查清单、Docker 一键部署链路和回归测试
- 构建稳定性：移除 `sass-embedded` 依赖，修复部分 macOS + Node 25 环境下前端 `vite build` 异常

## GitHub 提交说明简版

本次更新主要完善了自选股、数据同步、分析报告稳定性与公开提交准备：

- 自选股页面补齐港股实时展示与涨跌幅，优化“同步实时行情”的市场分流与失败回退逻辑
- 标签管理、快速标签、自选股编辑三处逻辑统一，新增标签自动入库、改名同步传播、删除后自动清理引用
- 新增同步历史与已同步数据清理能力，支持按类型查看与删除，并明确记录删除与真实数据删除的边界
- 新增“内容数据”页，可集中浏览单只股票的新闻、互动问答、社媒热度与新闻回退，并支持按来源、关键词和时间范围筛选
- A 股原生社媒同步从单一回退源扩展到多类官方互动和热度源，分析前预同步改为优先检查原生社媒数据
- “多数据源同步”页新增 A 股内容同步卡片，前端可以直接触发原生社媒和新闻回退同步，不再只能从自选股页进入
- A 股分析页不再继续显示“暂不支持社媒分析”的旧提示，改为明确说明会在分析前自动预同步原生社媒
- 调整研究经理与风险经理的决策提示词，避免把“持有”压缩成异常结论，并降低一般性风险被机械翻译成“卖出”的倾向
- A 股在线行情补齐增加独立 `sina_finance` 最后兜底，小批量缺口场景下会同时补齐价格、涨跌幅、交易日和更新时间
- A 股社媒分析报告新增来源分布、原生样本数和只剩新闻回退时的提示信息，便于判断结论是否需要重跑原生同步
- 港股原生新闻链路新增数据源尝试明细，统一新闻工具在原生源失败时会回显 `akshare / finnhub` 的尝试结果
- 完善单股同步的实时 / 非实时 / 混合数据源路由规则
- 配置管理页统一厂家与数据源的密钥来源展示，新增数据源密钥状态概览，并修复新增厂家时有效 API Key 被静默清空的问题
- 规范中文报告标题，增强长文本分析阶段稳定性
- 移除前端 `sass-embedded` 依赖，修复部分 macOS + Node 25 环境下的生产构建异常
- 修复 Docker 公开部署路径，统一运行时配置来源，补齐一键启动、健康检查和默认初始化流程
- 补充公开部署脚本、提交检查清单与回归测试，便于后续公开提交与部署
- 清理公开示例中的默认口令/长得像真实密钥的样例，降低公开提交时的误判风险
