# Changelog

## 记录口径

为避免埋没上游贡献，也避免把本仓库的独立工程化工作混写在一起，后续变更建议按以下口径记录：

- `backfill`
  - 表示参考上游 `TauricResearch/TradingAgents` 或其后续演进思路，在本仓库中做“局部回补”。
  - 这类改动通常保留本仓库现有架构，不追求逐文件同步上游实现。
- `feat`
  - 表示本仓库新增的独立功能或交互能力，例如中文化增强、部署脚本、数据清理、标签管理等。
- `fix`
  - 表示本仓库已有链路上的缺陷修复或稳定性修正。
- `docs`
  - 表示项目谱系说明、公开提交边界、部署与维护手册等非功能性文档更新。

建议在提交说明、PR 描述和后续 changelog 条目中保持这一分类习惯，以便清楚区分：

- 哪些能力来自上游思路回补
- 哪些能力属于 `TradingAgents-CN` 的本地增强
- 哪些改动只是问题修复或工程化整理

### 修复
- 修复新增厂家时有效 `API Key / API Secret` 被后端静默清空的问题，新增与编辑现在采用一致的校验与保存逻辑。
- 修复公开示例与本地启动脚本中的默认弱口令样例，统一替换为明显占位符，避免公开提交时引发误判。
- 修复“删除已同步数据”查询接口曾出现的 `404 / 422` 问题。
- 修复港股在自选股页“有价格但无涨跌幅”的问题，港股实时行情链路现在会正确返回 `change_percent / pct_chg`。
- 修复自选股“同步实时行情”对港股错误复用 A 股实时同步链路的问题。
- 修复港股实时刷新在外部源限流时直接判失败的问题，刷新失败后会尽量回退到最近缓存。
- 修复标签删除后，自选股中仍残留已删除标签的问题。
- 修复标签管理、快速标签、自选股编辑三处标签写入逻辑不一致的问题：
  - 编辑时新增标签会同步进入标签库
  - 标签改名会同步更新已挂载股票记录
  - 单只股票标签会自动去重
- 修复中文报告里残留英文标题的问题，中文偏好下报告标题会尽量规范为中文。
- 修复部分长文本分析阶段更容易超时的问题，对部分节点增强了流式处理与错误可观测性。

### 行为变更
- 删除自选股时可同时清理该股票相关的同步历史与已同步数据。
- “删除同步记录”和“删除真实已同步数据”保持解耦：
  - 删除记录不会自动删除真实数据
  - 删除真实数据不会自动删除同步记录
- 标签删除后会同步从股票记录中移除该标签，避免出现幽灵标签。
